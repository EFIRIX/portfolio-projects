# 💰 Финансовый копайлот

Telegram-бот, который избавляет от ручной категоризации трат: загрузил
CSV-выписку из банка — получил разбор по категориям, еженедельный отчёт со
сравнением к прошлой неделе и 1–2 содержательных наблюдения, привязанных к
конкретным цифрам.

> Для тех, кто ведёт личные финансы вручную в Excel/заметках и хочет
> автоматизации без тяжёлых приложений.

---

## Ключевое архитектурное решение: правила → LLM fallback

Категоризация идёт **в два этапа**, и это не оптимизация «на потом», а
осознанное проектное требование:

```
CSV → parse_csv → [ЭТАП 1: RulesCategorizer] → остаток → [ЭТАП 2: LLMCategorizer] → SQLite
                   словарь ключевых слов          ~20%      батчи по 20–30 транзакций
                   (config/category_rules.yaml)             (только то, что не разобралось)
```

- **`rules_categorizer.py`** — детерминированный словарь ключевых слов из
  `config/category_rules.yaml`. Без сети, без затрат, мгновенно. Покрывает
  **≥80%** транзакций тестового файла.
- **`llm_categorizer.py`** — вызывается **только** для остатка, который не
  разобрали правила. Запросы идут **батчами** (`LLM_BATCH_SIZE`, 20–30
  транзакций за раз), а не по одному запросу на строку.
- **`pipeline.py`** — связывает оба этапа в требуемом порядке. Оба модуля
  реализуют общий интерфейс `categorizer/base.py:Categorizer`, поэтому этапы
  взаимозаменяемы и тестируются по отдельности.

Зачем так: LLM — это деньги, задержка и недетерминизм. 80%+ обычных трат —
это одни и те же мерчанты («Пятёрочка», «Яндекс Такси», «ЖКХ»), которые
надёжнее и дешевле ловить словарём. LLM оставлен на действительно неизвестное.

Если LLM отключён (`LLM_PROVIDER=none`) или недоступен — бот не падает:
неразобранный остаток уходит в категорию **«прочее»**.

## 🔒 Что НЕ логируется и НЕ сохраняется

Требование безопасности реализовано в `security.py` и `csv_parser.py`:

| Данные | Что с ними происходит |
|---|---|
| **Номера карт (PAN)** | Вырезаются из описания (`[карта]`), в БД не пишутся |
| **Хвосты карт** (`*1234`, `**** 1234`) | Маскируются |
| **Номера счетов** (20 цифр) | Вырезаются (`[счёт]`) |
| **ФИО контрагентов** (`Иванову И. И.`) | Заменяются на `[контрагент]` |
| Прочие колонки выписки (номер карты/счёта как отдельные поля) | Парсер их вообще **не переносит** в модель — берутся только дата, сумма, описание, валюта |

Дополнительно на корневой логгер вешается `SensitiveDataFilter`
(`security.install_log_filter`) — последний рубеж, который маскирует любые
просочившиеся PAN/счета уже в тексте логов.

В SQLite попадает только **очищенное** описание операции.

## 🧱 Стек

Python 3.9+ · [aiogram 3.x](https://docs.aiogram.dev) · pandas · SQLite ·
YandexGPT / GigaChat через абстрактный интерфейс `LLMProvider`.

Смена провайдера — одна переменная `LLM_PROVIDER` в `.env`. Добавить новый
провайдер — реализовать `llm/base.py:LLMProvider.complete()`.

## 📈 Пример работы

После загрузки `examples/tinkoff_sample.csv` бот отвечает сводкой:

```
✅ Обработано операций: 20 (новых: 20)
📖 Правилами: 85%
Категории:
• еда: 5 шт.
• транспорт: 3 шт.
• жильё: 2 шт.
...
```

Команда `/report` (реальный вывод на тестовых данных, без LLM — наблюдения
строятся детерминированно и всё равно ссылаются на конкретные цифры):

```
📊 Отчёт за неделю 22.06–28.06.2026

• жильё: 45000 ₽ (1 шт.)  🔺 +733%
• здоровье: 4770 ₽ (2 шт.)  🆕
• еда: 3711 ₽ (5 шт.)  🔺 +214%
• транспорт: 2990 ₽ (3 шт.)  🆕
• развлечения: 1500 ₽ (1 шт.)  🔻 -25%
• подписки: 898 ₽ (2 шт.)  🆕

💰 Всего: 58869 ₽ (-25% к прошлой неделе)

🧠 Наблюдения:
— Больше всего вы потратили на «жильё» — 45000 ₽ за 1 операцию.
— Расходы на «жильё» выросли на 733% (5400 ₽ → 45000 ₽).
```

## 🏦 Поддержка разных банков (гибкий маппинг колонок)

Названия колонок в выписках банков различаются, поэтому они **не зашиты в код**,
а описаны в `config/bank_formats.yaml`. Парсер сам определяет формат по
заголовку файла (первый формат, все обязательные колонки которого найдены).

Добавить новый банк — скопировать блок в `bank_formats.yaml` и указать
соответствие колонок. Код менять не нужно:

```yaml
форматы:
  мой_банк:
    delimiter: ";"
    decimal: ","
    date_format: "%d.%m.%Y"
    columns:
      date: "Дата транзакции"
      amount: "Сумма в рублях"
      description: "Получатель"
```

В `examples/` лежат два готовых формата: `tinkoff_sample.csv` (`;`, запятая
как десятичный разделитель) и `sber_sample.csv` (`,`, точка).

## 📁 Структура

```
finance-copilot/
├── config/
│   ├── category_rules.yaml     # словарь ключевых слов (этап 1)
│   └── bank_formats.yaml       # маппинг колонок банков
├── examples/                   # тестовые CSV двух форматов (вымышленные данные)
├── src/finance_copilot/
│   ├── csv_parser.py           # гибкий парсинг + очистка PAN/ФИО
│   ├── security.py             # sanitize + фильтр логов
│   ├── db.py                   # SQLite (хранит только очищенное)
│   ├── report.py               # недельный отчёт + наблюдения
│   ├── categorizer/
│   │   ├── base.py             # интерфейс Categorizer
│   │   ├── rules_categorizer.py  # ЭТАП 1: правила
│   │   ├── llm_categorizer.py    # ЭТАП 2: LLM батчами
│   │   └── pipeline.py         # оркестрация правила → LLM
│   ├── llm/
│   │   ├── base.py             # абстрактный LLMProvider
│   │   ├── yandex.py           # YandexGPT
│   │   └── gigachat.py         # GigaChat
│   └── bot/                    # aiogram: main, handlers, keyboards
└── tests/                      # pytest: правила + парсер + защита данных
```
